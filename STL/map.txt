Example
#include <iostream>
#include <map>
using namespace std;

void example_map() {
    map<string, int> m;
    m["apple"] = 3;
    m["banana"] = 2;

    for (auto [key, val] : m)
        cout << key << " => " << val << endl;

    m.erase("banana");
    cout << "After erase:\n";
    for (auto [key, val] : m)
        cout << key << " => " << val << endl;
}

2.4 Multimap

Like map, but allows duplicate keys.

ðŸ—ºï¸ UNORDERED_MAP
Overview

An unordered_map is a hash table-based associative container that stores key-value pairs with unique keys and allows average O(1) lookups.
Unlike map, it does not maintain sorted order of keys.

Header:
#include <unordered_map>

Key Characteristics

Stores data as keyâ€“value pairs.

Keys are unique; values can repeat.

No ordering â€” elements stored in arbitrary order.

Average O(1) insertion and access.

Ideal for frequency counting, hash-based lookups, etc.

Example
#include <iostream>
#include <unordered_map>
using namespace std;

void example_unordered_map() {
    unordered_map<string, int> mp;

    // Insert elements
    mp["apple"] = 5;
    mp["banana"] = 10;
    mp.insert({"orange", 7});

    // Access
    cout << "Apple count: " << mp["apple"] << endl;

    // Find
    if (mp.find("banana") != mp.end())
        cout << "Banana found\n";

    // Iterate
    cout << "All elements:\n";
    for (auto &p : mp)
        cout << p.first << " -> " << p.second << endl;

    // Erase
    mp.erase("orange");

    cout << "Size after erase: " << mp.size() << endl;
}
